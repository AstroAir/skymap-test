# 每日知识交互体验优化设计（连续阅读模式）

## 背景与目标

- 当前每日知识已具备多源内容、筛选、收藏、历史与自动弹窗等基础能力。
- 本次优化聚焦“内容浏览体验”，不改动内容来源与业务策略。
- 目标是提供混合阅读模式：桌面端默认翻页、移动端默认列表流，同时保留现有筛选与操作链路。

## 设计边界

- 不改动 `lib/services/daily-knowledge/*` 的数据聚合逻辑。
- 不改动自动弹窗规则与设置项语义。
- 不引入重型动效和手势体系，优先稳定可用与低风险迭代。

## 方案选择

已评估三类方案（轻量增强 / 中度重构 / 体验优先重构），本次采用“轻量增强”：

- 在现有 `DailyKnowledgeDialog` 上增加视图模式与交互增强。
- 复用现有 store action（`next/prev/random/setCurrentItemById`）。
- 保持现有 UI 能力不回退，新增能力可渐进上线。

## 架构设计

### 状态设计

- 新增阅读 UI 偏好状态：
  - `viewMode: 'pager' | 'feed'`
  - `wheelPagingEnabled: boolean`
- 默认策略：
  - 桌面端默认 `pager`
  - 移动端默认 `feed`
- 建议持久化 UI 偏好（独立键或并入现有 store），仅影响交互展示，不影响内容业务状态。

### 组件结构

建议在 `components/starmap/knowledge/daily-knowledge-dialog.tsx` 轻拆分：

1. `DailyKnowledgeToolbar`
- 保留搜索/筛选/收藏筛选/结果数。
- 新增视图切换（Pager / Feed）。
- 在桌面 + Pager 时显示滚轮翻页开关。

2. `DailyKnowledgePagerView`
- 单条内容聚焦展示（沿用当前详情布局）。
- 显式上一条/下一条/随机操作。
- 支持键盘左右键翻页。
- 支持可开关滚轮翻页（含节流防误触）。

3. `DailyKnowledgeFeedView`
- 连续卡片流展示 `filteredItems`。
- 点击卡片设置当前项，保持与收藏/分享/跳转一致行为。

4. `DailyKnowledgeActions`
- 收藏、复制、分享、来源外链、相关天体跳转动作复用。

## 数据流设计

- 统一数据输入仍为 `filteredItems`。
- `effectiveItem` 继续作为全局当前项。
- `pager` 通过 `next/prev/random` 驱动当前项变化。
- `feed` 通过 `setCurrentItemById` 同步当前项。
- 切换视图不触发重新拉取，不重置筛选条件。

## 交互与可用性

### 桌面端

- 主交互：按钮 + 键盘左右键。
- 辅助交互：滚轮翻页（默认可开关）。
- 滚轮翻页需节流（建议 250-350ms）以降低误触与连续跳页风险。

### 移动端

- 默认列表流阅读。
- 保持滚动优先，不启用滚轮翻页逻辑。

### 兼容行为

- 当筛选结果不含当前项时，回退到首个可见项（延续现有逻辑）。
- `loading/error/noResults` 展示逻辑保持不变。

## 错误处理与鲁棒性

- 事件监听（键盘/滚轮）仅在 `open && viewMode==='pager'` 时注册。
- 关闭弹窗或切换到 `feed` 时必须移除监听，避免泄漏和误触。
- 相关天体跳转、复制分享、外链打开沿用现有错误提示逻辑。

## 测试设计

### 单元/组件测试（Jest + RTL）

- 桌面默认进入 `pager`，移动默认进入 `feed`。
- 视图切换可用，且筛选条件不丢失。
- `pager` 模式下键盘左右键触发 `prev/next`。
- 滚轮开关关闭不翻页，开启后翻页且节流生效。
- `feed` 点击卡片可更新当前项。

### E2E（Playwright）

- 桌面场景首次打开每日知识，默认展示翻页模式。
- 移动视口场景默认列表流，点击卡片后当前项联动正常。

## 验收标准

- 连续阅读效率提升：桌面翻页与移动列表流均可顺畅使用。
- 不破坏收藏、历史、筛选、分享、相关天体跳转链路。
- 自动弹窗与内容来源行为与现状一致。
- 相关新增/回归测试通过。

## 后续扩展（不在本次范围）

- 更强的动效翻页与拖拽手势。
- 阅读进度、重点高亮、长文折叠。
- 更深度的个性化排序与推荐。
